# 🗺 Дорожная карта Tempo (Roadmap)

Стратегия превращения музыкального плеера в полноценную SaaS-платформу.

## 🚀 Фаза 1: Автоматизация биллинга (Q2 2026)
- **Интеграция Платежей:** Подключение Stripe/CloudPayments.
- **Webhooks:** Автоматическое обновление `track_limit` и `subscription_tier` при поступлении оплаты.
- **Billing UI:** Страница "Подписка" в профиле пользователя, где видно текущий лимит и кнопку "Улучшить тариф".

## 📦 Фаза 2: Расширение Админки
- **Dashboard 2.0:** Графики роста пользователей и выручки.
- **Audit Logs UI:** Полноценный экран просмотра логов (таблица `audit_logs` уже создана в БД).
- **Track Management:** Возможность админу удалять/редактировать любые треки прямо из таблицы.

## 💎 Фаза 3: Новые идеи для монетизации
1. **Offline Pro Mode:** Расширенное хранение треков в IndexedDB (PWA) для работы в залах с плохим интернетом (только для платных аккаунтов).
2. **AI Stem Splitting:** Удаление вокала или выделение ударных/баса из трека (интеграция с API для разделения аудио).
3. **Training Presets Marketplace:** Платные пакеты тренировок (наборы треков с автоматической сменой BPM по таймеру).
4. **Studio Accounts:** Групповые подписки для танцевальных школ (1 владелец + N тренеров).

## 🛠 Фаза 4: Архитектурная чистота
- Полная декомпозиция `PlayerUI.tsx`.
- Автоматический прогон Vitest и Jest в GitHub Actions (уже настроено в v1.0.47).
- Миграция бэкенда на TypeScript.
