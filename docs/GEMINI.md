# ⚡️ Gemini Context: Tempo Project (v2.2)

## 🛑 АВАРИЙНЫЙ ПРОТОКОЛ
**ГЛАВНОЕ ПРАВИЛО:** Ветка `main` заблокирована для мусора. Любой пуш должен иметь обновленный `CHANGELOG.md` и версию.

### 1. Как не сломать CI/CD
Если ты меняешь версию в `constants.tsx`, ты ОБЯЗАН обновить `CHANGELOG.md`. 
Скрипт `scripts/check-version.js` стоит на страже и уронит билд при расхождении.

### 2. Структура Проекта
- **Логика:** `server/controllers/`
- **Маршруты:** `server/routes/`
- **Фронт:** `App.tsx` + `hooks/`
- **Стили:** Дизайн-система описана в `DESIGN_SYSTEM.md`.

### 3. Автоматизация Changelog
При деплое GitHub Action сам парсит `CHANGELOG.md` и делает `INSERT` в таблицу `changelogs`. Не нужно делать это руками.

### 4. Порядок работы
1. Кодим в ветке `dev`.
2. Тестируем на порту **3006** (Test).
3. Когда всё ОК -> Обновляем версию и ченджлог -> Мержим в `main`.
4. GitHub сам доставит код на ПРОД (3005).
