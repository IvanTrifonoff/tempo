# Операции и Тестирование (Ops & Testing)

## 🚀 CI/CD Пайплайн
Деплой полностью автоматизирован через GitHub Actions (`.github/workflows/deploy.yml`).

### Как выпустить релиз:
1. Внесите изменения в код.
2. Обновите версию в `constants.tsx` (например, `1.0.35`).
3. Добавьте секцию `## [1.0.35]` в начало `CHANGELOG.md` с описанием изменений.
4. Сделайте `git push origin main`.
5. GitHub сам:
   - Проверит версию.
   - Соберет образ.
   - Запушит в GHCR.
   - Обновит контейнеры на сервере.
   - Запишет ченджлог в базу данных.

## 🗄 База Данных (Миграции)
- **Создать:** `node-pg-migrate create name_of_migration` (в папке `server`).
- **Накатить локально:** `npm run migrate up`.
- **На сервере:** Накатываются автоматически при старте контейнера.

## 🐳 Локальная разработка
Для запуска проекта с "билдом" (как на проде):
`sudo docker compose up -d --build`

Для просмотра логов:
`sudo docker logs -f tempo-app` (Prod)
`sudo docker logs -f tempo-test-pg` (Test)

## 🧪 Валидация
Вы можете запустить проверку версий вручную:
`node scripts/check-version.js`
