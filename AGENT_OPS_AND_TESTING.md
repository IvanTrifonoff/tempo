# Эксплуатация и Тестирование (Ops & QA)

Этот файл содержит инструкции по обслуживанию системы, работе с данными и ручному тестированию.

## 1. Управление Данными (Backup & Restore)

Так как база данных (`db.json`) и файлы (`uploads`) хранятся в Docker Volumes, их нужно бекапить вручную.

### Создание резервной копии (Backup)
Выполнять на сервере (`82.202.141.81`):
```bash
# Создать архив с данными и файлами
sudo docker run --rm --volumes-from tempo-app -v $(pwd):/backup ubuntu tar cvf /backup/tempo_backup_$(date +%Y%m%d).tar /app/server/data /app/server/uploads
```
*Для тестового контура используйте `--volumes-from tempo-test`.*

### Восстановление (Restore)
**Внимание:** Это перезапишет текущие данные!
```bash
# 1. Остановить контейнер
sudo docker stop tempo-app

# 2. Восстановить данные
sudo docker run --rm --volumes-from tempo-app -v $(pwd):/backup ubuntu tar xvf /backup/tempo_backup_YYYYMMDD.tar

# 3. Запустить контейнер
sudo docker start tempo-app
```

## 2. Инвентаризация Секретов (Security Audit)

На данный момент секреты **захардкожены** в коде. При смене паролей или утечке, править нужно в следующих файлах:

| Секрет | Файл | Строка (прим.) | Описание |
| :--- | :--- | :--- | :--- |
| **JWT_SECRET** | `server/server.js` | `const JWT_SECRET = ...` | Ключ подписи токенов авторизации. |
| **SMTP_PASS** | `server/email.js` | `pass: '...'` | Пароль приложения для `tempo@trfnv.ru` (Mail.ru). |
| **SMTP_USER** | `server/email.js` | `user: '...'` | Логин почты отправки. |

*Задача для следующего разработчика: Перенести эти значения в `.env` и пробрасывать в Docker контейнер.*

## 3. Чек-лист ручного тестирования (QA Manual)

Так как автотестов нет, перед каждым релизом проходи этот чек-лист:

### A. Ролевая модель
1.  [ ] **Admin:** Войти -> Увидеть кнопку "Пользователи" (Щит) -> Удалить тестового юзера.
2.  [ ] **Coach:** Регистрация -> Получение Email -> Переход по ссылке -> Активация -> Вход.
3.  [ ] **Student:** Переход по инвайт-ссылке тренера -> Форма "Регистрация ученика" -> Вход -> Видимость ТОЛЬКО треков тренера.

### B. Аудио (Самое сложное)
*Тестировать на реальном iPhone (Safari) и Android (Chrome).*
1.  [ ] **Фон:** Включить трек -> Свернуть приложение -> Музыка должна играть.
2.  [ ] **Lock Screen:** Заблокировать телефон -> Должны быть видны кнопки паузы/переключения и название трека.
3.  [ ] **Coach Mode:** Включить "Свисток" -> Включить "Хлопки" -> Хлопнуть дважды -> Музыка должна встать на паузу.
4.  [ ] **Выключение:** Выключить "Свисток" -> Свернуть приложение -> Музыка должна играть (проверка отключения AudioContext).

### C. PWA
1.  [ ] **Install:** Проверить появление плашки "Установить приложение" (или через меню браузера).
2.  [ ] **Offline:** Выключить интернет -> Приложение должно открываться, закешированные треки играть.
3.  [ ] **Update:** При деплое новой версии -> Появление желтого уведомления "Update" -> Перезагрузка.

## 4. Экстренные ситуации (Troubleshooting)

*   **Сервер упал (502):**
    `sudo docker logs tempo-app --tail 50` — ищи SyntaxError или ошибки подключения к БД.
*   **Письма не уходят:**
    `sudo docker logs tempo-app | grep SMTP` — если ошибка авторизации, обнови пароль приложения в Mail.ru.
*   **Музыка не играет (Format Error):**
    Проверь, что в БД путь к файлу начинается с `/uploads/` и файл физически существует в томе `tempo-uploads`.
