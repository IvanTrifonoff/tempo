# Глубокое погружение (Deep Dive) — Архитектура

## 🛠 Выбор Стека
1. **PostgreSQL:** Выбран для обеспечения целостности данных (Foreign Keys) и поддержки транзакций. Это критично при росте количества плейлистов и связей между треками.
2. **node-pg-migrate:** Инструмент для версионирования схемы. Позволяет делать `up` и `down` (откат) изменений БД.
3. **Winston:** Используется для структурированного логирования. Позволяет легко интегрироваться с внешними системами сбора логов (ELK/Loki).
4. **Compression (Gzip):** Внедрено на уровне Express для ускорения загрузки статики (JS бандл сжат с 320КБ до ~80КБ).

## 🔒 Безопасность
- **JWT:** Токены содержат `id`, `role` и `coachId`.
- **ErrorHandler:** Централизованный middleware скрывает детали ошибок (stack trace) в режиме `production`.
- **Docker Security:** Приложение работает под пользователем `node` (UID 1000). Папки `uploads` защищены правами доступа.

## 🎧 Аудио и iOS (The Critical Part)
iOS блокирует автоматический запуск аудио. 
**Решение:** Хук `useAudioContext` лениво инициализирует `AudioContext` только когда пользователь нажимает "Play". Мы используем один контекст для метронома и для музыки, чтобы они были идеально синхронизированы.

## 📈 Оптимизация SQL
Для получения плейлистов используется один запрос с `json_agg`, что решает проблему N+1. Это позволяет загружать 100+ плейлистов за один цикл обращения к БД.
