# Контекст разработки Tempo.TRFNV (Agent Handoff)

**Актуальная версия:** 1.0.32
**Статус:** PROD (Stable)

## 1. Архитектура Данных (db.json)

База данных — это единый JSON файл. При разработке новых фич учитывай схему:

### Users
*   `role`: 'admin', 'coach', 'student'.
*   `coachId`: (только для учеников) содержит `id` тренера.
*   `isVerified`: (только для тренеров) флаг активации через email.
*   `verificationToken`: временный токен для ссылки из письма.
*   `favorites`: массив `id` треков.

### Tracks
*   `ownerId`: `id` пользователя, который загрузил трек.
*   `isPublic`: если `true`, трек видят все. Если `false`, видят только владелец и его ученики.

## 2. Логика Доступа (Backend)

### Фильтрация контента (`GET /api/tracks`):
*   **Гость:** Только `isPublic: true`.
*   **Student:** `isPublic: true || ownerId === user.coachId`.
*   **Coach:** `isPublic: true || ownerId === user.id`.
*   **Admin:** Все треки без исключения.

### Регистрация и Инвайты:
*   Инвайт-ссылка имеет формат `/?invite=COACH_ID`.
*   При регистрации с `inviteCode`, сервер ищет тренера с таким ID. Если находит — создает пользователя с `role: student` и привязывает `coachId`. Такие пользователи активируются автоматически.

## 3. Аудио-движок (Critical Knowledge)

**Проблема iOS Safari:** Если создать `AudioContext` при старте страницы, iOS "засыпает" при сворачивании браузера.
**Решение:** 
1.  `<audio>` элемент инициализируется нативно. 
2.  `AudioContext` создается **только** внутри `useCallback` (функция `initAudioCtx`), который вызывается при нажатии Play, если включен метроном или детекция хлопков.
3.  Используется `latencyHint: 'playback'`. 
4.  Если спец-функции (хлопки/метроном) не нужны, `AudioContext` вообще не должен создаваться — это гарантирует работу музыки в фоне на 100%.

## 4. Сборка и Tailwind (v3)

Проект переведен с CDN на локальную сборку.
*   **Конфиг:** `tailwind.config.js` сканирует `App.tsx`, `components/` и `constants.tsx`.
*   **Динамические цвета:** Цвета жанров прописаны в `constants.tsx` (объект `STYLE_COLORS`). Они должны быть в формате полных классов (напр. `bg-rose-500`), чтобы PurgeCSS их не удалил.

## 5. Технический долг (Задачи для следующего агента)

1.  **JWT Secret:** Сейчас `JWT_SECRET` захардкожен в `server.js`. Нужно вынести в `.env`.
2.  **Email Config:** Данные SMTP Mail.ru захардкожены в `email.js`. Нужно вынести в `.env`.
3.  **Error Handling:** На фронтенде многие `fetch` не имеют обработки `catch` с выводом ошибки пользователю (сейчас только `console.error`).
4.  **Admin UI:** Добавить возможность редактировать список плейлистов других пользователей.
5.  **Refactoring:** Логика модальных окон в `App.tsx` стала слишком громоздкой, стоит вынести их в папку `components/modals/`.

## 6. Команды управления на сервере

Если что-то сломалось, проверь SMTP:
`sudo docker logs tempo-app | grep SMTP`

Ручная миграция базы:
Сервер запускает миграцию ролей автоматически при старте. Чтобы принудительно обновить данные, просто перезапусти контейнер.
