# История изменений (Changelog) — tempo.TRFNV

## [1.0.34] — 2025-12-31
### 🏗 Архитектурная стабилизация
- **Modular Backend:** Полная декомпозиция `server.js`. Код разбит на контроллеры, роуты и middleware. Это база для будущего расширения.
- **Database Migrations:** Внедрена система миграций `node-pg-migrate`. Теперь изменения структуры БД версионируются и применяются автоматически.
- **Logging 2.0:** Внедрен Winston для структурированного логирования в формате JSON. Ошибки теперь фиксируются в `error.log`.
- **Performance:** Включено Gzip-сжатие на сервере. Время загрузки статики сокращено в 3-4 раза.
- **Cache-Control:** Настроено агрессивное кэширование JS/CSS бандлов (1 год), что делает повторные открытия приложения мгновенными.

### 🛡 Безопасность и Ops
- **Centralized Error Handling:** Все ошибки API обрабатываются единым middleware, предотвращая утечку данных о сервере.
- **Docker Cleanup:** На сервере настроена автоматическая очистка мусора Docker по расписанию.
- **Documentation:** Полное обновление инструкций для ИИ-агентов, актуализирована карта проекта.

## [1.0.33] — 2025-12-31
### 🏗 Архитектура и Производительность
- **PostgreSQL:** Полный переход с JSON-файла на реляционную базу данных. Обеспечена атомарность операций и сохранность данных.
- **SQL Optimization:** Устранена проблема N+1 запросов при загрузке плейлистов. Теперь списки загружаются одним оптимизированным запросом.
- **Refactoring:** Логика аудио-движка и плеера вынесена в кастомные хуки (`usePlayer`, `useAudioContext`), что облегчило поддержку кода.

### 🛡 Безопасность
- **Non-root Docker:** Приложение в контейнере теперь работает от пользователя `node`, убраны права 777.
- **Secrets Management:** Все учетные данные (SMTP, БД) вынесены в переменные окружения и убраны из исходного кода.

### 🎨 Интерфейс (UI/UX)
- **Edit Modal:** Инлайн-редактирование заменено на полноценное модальное окно.
- **Design System:** Унифицированы стили кнопок, инпутов и скруглений согласно дизайн-системе. Исправлены цвета скроллбаров.
- **Metronome:** Звук метронома заменен на приятный "Woodblock" (синтез).
- **Changelog:** Добавлено окно "Что нового", которое загружает список изменений динамически с сервера.

## [1.0.32] — 2025-12-30
### 🔐 Система ролей и Доступ
- **SaaS Архитектура:** Реализовано разделение на три роли: **Администратор**, **Тренер** и **Ученик**.
- **Верификация:** Добавлена полноценная регистрация тренеров с подтверждением через Email (SMTP Mail.ru).
- **Инвайты:** Реализована система приглашений. Ученики регистрируются по уникальной ссылке тренера и автоматически привязываются к его аккаунту.
- **Приватность:** Умная фильтрация контента. Ученики видят только общие треки и личную библиотеку своего тренера.

### 🛠 Управление и Администрирование
- **User Management:** Создана отдельная панель управления пользователями для админа (смена ролей, удаление, ручная активация).
- **Inline Editing:** Добавлена возможность редактирования названий треков и имен исполнителей прямо в списке (для владельцев контента).
- **Metadata Fix:** Исправлен баг, при котором плейлисты не удалялись из базы данных.

### 🎧 Аудио и Производительность
- **Background Play 2.0:** Глубокая оптимизация аудио-движка для iOS. Переход на ленивую инициализацию `AudioContext` и режим `playback` позволил добиться стабильной работы в фоне.
- **Instant Start:** Добавлена предзагрузка метаданных (`preload="metadata"`), музыка начинает играть мгновенно без ожидания полной загрузки файла.
- **Media Session:** Интеграция с системными контроллерами (управление треками с экрана блокировки и гарнитуры).

### 🎨 Интерфейс (UI/UX)
- **Реорганизация:** Настройки разделены на технические (шестеренка) и тренировочные (свисток).
- **Компактный вид:** Полностью переработана сетка треков и хедер. Приложение стало занимать на 30% меньше вертикального места без потери читаемости.
- **Visual Feedback:** Добавлен оверлей Play/Pause на карточки треков и индикатор метронома.
- **Иконки:** Внедрен профессиональный набор иконок Lucide.

### 🚀 Техническая часть
- **Zero Dependencies (CDN-free):** Tailwind CSS переведен с CDN на локальную сборку. Приложение теперь полностью автономно и работает быстрее.
- **CI/CD Flow:** Настроен конвейер деплоя на тестовый (`tempotest`) и основной сервера.
- **PWA Lifecycle:** Исправлена логика обновления приложения. Теперь при выходе новой версии пользователь получает уведомление и может обновиться одной кнопкой.
