# 🌐 API Specification

Все запросы должны начинаться с префикса `/api`. Большинство эндпоинтов требуют заголовок `Authorization: Bearer <token>`.

## 🔐 Authentication (`/auth`)
| Метод | Путь | Auth | Описание |
| :--- | :--- | :--- | :--- |
| `POST` | `/register` | No | Регистрация (тренер или студент по инвайту) |
| `POST` | `/login` | No | Авторизация и получение токена |
| `GET` | `/me` | Yes | Получение данных текущего пользователя |
| `GET` | `/verify` | No | (Root level) Верификация email по ссылке |

## 🎵 Tracks (`/tracks`)
| Метод | Путь | Auth | Описание |
| :--- | :--- | :--- | :--- |
| `GET` | `/` | Optional | Список доступных треков (с учетом ролей) |
| `POST` | `/` | Yes | Загрузка нового трека (Multer: `file` или `url`) |
| `PATCH` | `/:id` | Yes | Редактирование метаданных (только владелец/админ) |
| `DELETE` | `/:id` | Yes | Удаление трека и файла с сервера |
| `POST` | `/favorites` | Yes | Добавить/удалить из избранного (в роуте `/users`) |

## 📂 Playlists (`/playlists`)
| Метод | Путь | Auth | Описание |
| :--- | :--- | :--- | :--- |
| `GET` | `/` | Yes | Личные плейлисты пользователя |
| `POST` | `/` | Yes | Создание нового плейлиста |
| `DELETE` | `/:id` | Yes | Удаление плейлиста |
| `POST` | `/:id/tracks` | Yes | Добавление трека в плейлист |
| `DELETE` | `/:id/tracks/:trackId` | Yes | Удаление трека из плейлиста |

## 🛠 Admin (`/admin/users`)
| Метод | Путь | Auth | Описание |
| :--- | :--- | :--- | :--- |
| `GET` | `/users` | Admin | Список всех пользователей системы |
| `PATCH` | `/users/:id` | Admin | Изменение роли/статуса пользователя |
| `DELETE` | `/users/:id` | Admin | Полное удаление аккаунта |
