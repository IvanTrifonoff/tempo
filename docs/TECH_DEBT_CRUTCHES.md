# Технический долг и "Костыли" (Critical Technical Debt)

Этот документ предназначен для следующего разработчика/агента. Здесь описаны места, которые работают в "ручном" или "полуавтоматическом" режиме из-за конфликтов инфраструктуры.

## 🔴 Критические "Костыли"

### 1. Миграции БД (Disabled Auto-migrate)
**Проблема:** Библиотека `node-pg-migrate` на VPS конфликтует с именами файлов (смесь `.js` и `.cjs`) и порядком (timestamps). Контейнер уходил в вечный рестарт.
**Текущее решение:** 
- В `server/docker-entrypoint.sh` команда `npm run migrate` закомментирована.
- В `server/package.json` добавлен флаг `--check-order false`.
- Поля `track_limit`, `is_banned`, `subscription_tier` и таблица `audit_logs` созданы **вручную** через SQL.
**Что нужно сделать:** Выполнить "сброс" миграций (Migration Reset) — привести все файлы к `.cjs`, очистить таблицу `pgmigrations` и прогнать один чистый файл схемы.

### 2. Монолит `PlayerUI.tsx`
**Проблема:** В рамках борьбы с монолитом `App.tsx` был разделен на роутер и UI. Но `PlayerUI.tsx` всё еще содержит ~950 строк кода.
**Что нужно сделать:** Декомпозировать `PlayerUI.tsx` на мелкие компоненты: `Metronome.tsx`, `TrackList.tsx`, `PlayerControls.tsx`, `PlaylistSection.tsx`.

## 🟡 Средний приоритет

### 3. Дублирование кода загрузки
**Проблема:** `UploadTrackModal.tsx` (бывший `AdminPanel.tsx`) и контроллер `tracksController.js` имеют жестко заданные лимиты и логику, которая частично дублируется в тестах.
**Что нужно сделать:** Вынести логику проверки лимитов в отдельный `LimitService` на бэкенде.

### 4. Git на VPS
**Проблема:** Деплой сейчас идет через `scp` архивов, так как Git на сервере требует авторизации.
**Что нужно сделать:** Настроить Deploy Keys для GitHub на VPS, чтобы использовать `git pull` напрямую.

## 🟢 Низкий приоритет

### 5. Типизация Backend
**Проблема:** Бэкенд на JS, фронтенд на TS. Контракты API (особенно новые для админки) не синхронизированы через общие интерфейсы.
**Что нужно сделать:** Внедрить общие типы или использовать Zod-схемы как единственный источник правды.
