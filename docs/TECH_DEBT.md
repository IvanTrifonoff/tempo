# Технический долг и Риски (Technical Debt)

Документ создан: 31.12.2025
Статус проекта: v1.0.33 (Postgres Migration Phase)

## 🔴 Высокий Приоритет (Critical)

1.  **Монолитный `server.js`:**
    *   **Проблема:** Вся логика бэкенда (роутинг, бизнес-логика, работа с БД, конфигурация) находится в одном файле на 500+ строк.
    *   **Риск:** Низкая поддерживаемость, сложность тестирования, дублирование кода.
    *   **План:** Разбить на слои: `routes` (Express), `controllers` (HTTP логика), `services` (Бизнес-логика), `db` (SQL).

2.  **Отсутствие Миграций БД:**
    *   **Проблема:** Схема управляется вручную через `schema.sql`. Нет истории изменений, нет возможности отката (rollback).
    *   **Риск:** Рассинхронизация схемы между Dev, Test и Prod. Потеря данных при неудачных изменениях.
    *   **План:** Внедрить `node-pg-migrate`.

3.  **Отсутствие Мониторинга:**
    *   **Проблема:** Ошибки пишутся в `console.error` внутри контейнера. Нет алертинга.
    *   **Риск:** "Слепой полет" в продакшене.
    *   **План:** Подключить Sentry или хотя бы структурированное логирование (Winston/Pino).

## 🟡 Средний Приоритет (Medium)

1.  **Разрыв типов (JS Backend vs TS Frontend):**
    *   **Проблема:** Бэкенд возвращает `snake_case`, фронтенд ждет `camelCase`. Маппинг делается вручную. Нет общих интерфейсов.
    *   **Риск:** Ошибки типа `undefined` в рантейме при изменении API.
    *   **План:** В идеале — перевести бэкенд на TS или использовать Zod для валидации контрактов.

2.  **Ручной CI/CD:**
    *   **Проблема:** Деплой через скрипты локальной машины.
    *   **Риск:** Зависимость от локального окружения разработчика.
    *   **План:** Настроить GitHub Actions.

3.  **Отсутствие тестов Фронтенда:**
    *   **Проблема:** Логика плеера и аудио не покрыта тестами.
    *   **Риск:** Регрессии при рефакторинге AudioContext.

## 🟢 Низкий Приоритет (Low)

1.  **Single Thread Node.js:** Не используется кластеризация.
2.  **Hardcoded Strings:** Тексты ошибок и сообщений захардкожены в коде.
