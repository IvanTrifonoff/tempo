# Отчет о разработке проекта tempo.TRFNV

Этот документ содержит историю изменений, описание архитектуры и инструкции для следующего разработчика/агента.

## Обзор проекта
**Цель:** Приложение для тренеров бальных танцев с управлением темпом, автопилотом и детекцией хлопков.
**Стек:** React 19 (Vite), Express.js (Node.js), Docker, JWT Auth, Web Audio API.

## Выполненные задачи и инструменты

### 1. Архитектура (v1.0.0)
- **Переход на Client-Server:** Ранее проект был SPA. Реализован бэкенд на **Express.js**.
- **База данных:** Используется JSON-база (`db.json`) в папке `server/data`. Папка примонтирована через Docker Volume для сохранения данных.
- **Инструменты:** Node.js, Docker, Docker-compose (реализован через ручные команды запуска контейнера).

### 2. Авторизация и Безопасность
- **JWT + Bcrypt:** Полноценная регистрация и вход. Пароли хешируются.
- **Персонализация:** Плейлисты и Избранное привязаны к ID пользователя.
- **Ролевая модель:** Пользователи с email, содержащим `admin`, получают права на загрузку и удаление треков.
- **Инструменты:** `jsonwebtoken`, `bcryptjs`.

### 3. Модуль «Clap Control» (Управление хлопками)
- **Технология:** Web Audio API (`AudioContext`, `AnalyserNode`).
- **Маршрутизация:** Для стабильности на iOS музыка из `<audio>` заведена в тот же `AudioContext`, что и микрофон.
- **Алгоритм:** Двойной хлопок (Double Clap) в окне 800мс. Анализ высокочастотных транзиентов (2.5кГц - 8кГц) без сглаживания.
- **Режим тренера:** Wake Lock API предотвращает сон устройства.

### 4. PWA и Локализация
- **Vite PWA:** Настроено через `vite-plugin-pwa`. Стратегия обновления `autoUpdate`.
- **i18next:** Поддержка RU, EN, ES. Все строки, включая жанры танцев, локализованы.

## Известные проблемы и нюансы (Handover Notes)

1.  **iOS PWA Forms:** Были проблемы с фокусом и клавиатурой. Решено через принудительный `z-index: 9999`, шрифт `16px` (защита от зума) и упрощенную верстку.
2.  **Bluetooth на iOS:** Активация микрофона в Safari может переключать Bluetooth в режим HFP (низкое качество). Реализован "единый аудио-граф" для минимизации этого эффекта.
3.  **Deploy:** Деплой производится через упаковку папки в `.tar.gz`, отправку через `scp` и сборку `docker build` на сервере `82.202.141.81`.
